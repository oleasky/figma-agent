## Directory Purposes

**.planning/**
- Purpose: Project planning and codebase analysis
- Contains: `codebase/` subdirectory with mapping documents
- Key files: STACK.md, ARCHITECTURE.md, STRUCTURE.md, etc.
- Subdirectories: `codebase/`

**.vscode/**
- Purpose: VSCode editor configuration
- Contains: `settings.json` with workbench color customizations
- Key files: `settings.json`
- Subdirectories: None

## Key File Locations

**Entry Point:**
- `CLAUDE.md` - Project index: knowledge modules, skills, usage guide, CSS strategy

**Knowledge Modules:**
- `knowledge/README.md` - Naming conventions, module structure, planned modules list
- Knowledge files follow `{domain}-{topic}.md` naming (e.g., `figma-api-rest.md`)

**Skills:**
- `commands/figma/README.md` - Skill conventions, invocation guide, planned skills list
- Skills invoked via `/figma:skill-name` in Claude Code

**Archived Specs:**
- `archive/initial-agent.md` - Original spec (superseded)
- `archive/initial-agent-audit.md` - Audit of original spec (superseded)
- `archive/agent-rewritten-01.md` - Corrected v2 spec (superseded)

**Configuration:**
- `.vscode/settings.json` - VSCode theme colors
- `.env.example` - Safe placeholder values for Figma API
- `.gitignore` - Protects .env files, OS/editor artifacts

## Naming Conventions

**Files:**
- kebab-case.md for all documentation files
- Numbered suffix for versions: `agent-rewritten-01.md`

**Directories:**
- kebab-case (`.planning/`, `.vscode/`)

**Special Patterns:**
- Leading dot for config/planning directories (`.planning/`, `.vscode/`)
- UPPERCASE.md for planning documents (STACK.md, ARCHITECTURE.md, etc.)

## Where to Add New Content

**Knowledge Modules:**
- Location: `knowledge/{domain}-{topic}.md`
- Follow structure from `knowledge/README.md`

**Skills:**
- Location: `commands/figma/{skill-name}.md`
- Follow conventions from `commands/figma/README.md`

**Planning Documents:**
- Location: `.planning/` directory

## Special Directories

**.planning/**
- Purpose: Project planning and analysis documents
- Source: Generated by codebase mapping and project planning workflows
- Committed: Yes (planning artifacts)
